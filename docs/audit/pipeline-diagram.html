Evaluate — the oracle contract new this round

Judge any deliverable with cited evidence — not just web UIs. One verdict per criterion, evidence-or-FAIL, single judge untouched.

Dispatch — ba tags each row, the evaluator picks the probe

Test Surface row

mechanically derived

spec-evaluator dispatch

Reads oracle: and runs the matching probe via Bash. Absence of evidence = FAIL; never greps source in place of running it.

single judge

ui · default

Web app — Playwright a11y-tree loop (in-skill). Evidence: node, state before/after, console.

process

CLI / script — spawn with argv in a temp dir; grade exit + stdout/stderr + crash check.

test

Library / module — run its own suite; exit + failing names. Zero tests run = FAIL.

snapshot

Generator / pure refactor — diff observed stdout vs a golden (empty = PASS).

http

Service / API — start on a free port, poll ready, assert status + body/JSON, tear down. Unreachable = FAIL all.

The contract changes how evidence is gathered, never who decides. Each non-ui oracle is a self-contained spawn-and-grade procedure documented in spec-evaluator/references/probing.md (with a shared expect grammar) and backed by an executable dev/CI reference impl in scripts/oracles/ with a negative control proving it discriminates.

F9

Ship-safe skills — what installs vs. what's dev-only fixed this round

The grill that caught it: shipped guidance pointed at files that vanish on install. A skill must be self-contained.

Before → after

Before — dangling refs

probing.md told the agent to run node scripts/oracles/*.mjs and read examples/*.json / docs/audit/* — none of which ship. Bare relative paths also resolve against the user's cwd, not the plugin root.

F9

After — self-contained

probing.md describes each oracle as an inline procedure + expect grammar — zero script / example / doc paths. The runners + fixtures are demoted to dev/CI only.

Guard #12 makes it permanent. A new structural check fails any shipped skill file that cites a repo-only path (scripts/, examples/, docs/audit·plan·research/, tests/); runtime paths the harness itself creates (docs/shapeup-sdlc/, .shapeup-sdlc/) stay allowed. A mutation test — inject a dangling ref → suite goes red, revert → green — confirms the guard catches the class, closing the false confidence that the cwd-dependent oracle checks had given.

Migration / update

migrate.sh — versioned runner (Flyway/Rails-style): updates code (replace skills) then migrates data by applying pending scripts/migrations/NNNN__*.sh in order, tracked in a committed .harness-migrations ledger. Idempotent; each future version adds its own migration.

Read the map honestly. Everything green runs today. The eval-gate is a deliberate placeholder, and the behavioral evals it will one day enforce (Tier-1/Tier-2) were documented as "landed" but were never committed — they are the next real work, and the only part of the pipeline that needs a Claude fresh-login.
