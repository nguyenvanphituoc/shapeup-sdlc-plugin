Migration / update

migrate.sh — versioned runner (Flyway/Rails-style): updates code (replace skills) then migrates data by applying pending scripts/migrations/NNNN__*.sh in order, tracked in a committed .harness-migrations ledger. Idempotent; each future version adds its own migration.

Read the map honestly. Everything green runs today. The eval-gate is a deliberate placeholder, and the behavioral evals it will one day enforce (Tier-1/Tier-2) were documented as "landed" but were never committed — they are the next real work, and the only part of the pipeline that needs a Claude fresh-login.
